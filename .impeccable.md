## Validation cadence (authoritative)

For all design-validation questions — "should we show this to testers",
"do we need a user study", "who signs off" — defer to
[docs/validation-philosophy.md](./docs/validation-philosophy.md). That
doc supersedes any "Gate 2 walkthrough" or "Cúpula DT labeler session"
language in older planning docs. Four signals, in order of preference:

1. Automated (tests, telemetry, visual regression, lint)
2. Self-validation (owner-run via dev-browser + screenshots)
3. Informal friends circle when demo-ready
4. Target-community release only after 1-3 have cleared

Do not propose external human validation as a gate for design polish.

## Users

**Who.** Flesh and Blood (FaB) TCG players in the closed Cúpula DT
community (~47 people, Pelotas/RS, Brazil). Mostly competitive amateurs
who already own a physical card collection and track multiple decks.

**Context.** Desktop-first at home for deep deck planning; mobile during
in-person play sessions at local stores and cafés (Gate 2 walkthroughs
happen presencially, not remote). Brazilian Portuguese speakers but app
copy is English today — a concession, not a preference.

**Job-to-be-done.** "What can I play right now given the cards I already
own?" The substitution engine's answer is the entire value prop: a
tracked deck shows how close it is to playable, surfaces what's missing,
and recommends proxies from the user's collection instead of forcing
purchases. The UI is judged by whether a new user can understand this
mental model on first contact, without explanation.

## Brand Personality

Three words: **arcane, tactical, artisanal**.

- **Arcane** — the world of Rathe is high-fantasy occult. Brass, oxblood,
  parchment, gothic serifs. Decorative accents (◆, roman numerals, hand-
  set type) belong. Neon, cyan, pastel gradients do not.
- **Tactical** — this is a decision tool, not a store. Numbers and
  readiness states are the primary content. Visual noise competes with
  comprehension; restraint wins.
- **Artisanal** — signature details are rare and purposeful, not
  sprinkled. The brass readiness display is reserved for one metric. The
  deckbox SVG is hand-tuned, not a stock icon. Where templates would add
  a card-with-icon-and-rounded-corners, we reach for something the
  project actually needs instead.

## Aesthetic Direction

**Theme.** Dark is the primary surface and the only theme shipped in
Plan A. Light tokens exist but are not yet tone-corrected (Plan C). Do
not optimize for light-theme appearance in Plan A deliverables.

**Palette** (from `apps/web/src/styles/tokens.css`):
- Canvas `#0c0d10`, surface `#15171c`
- Brass accent `#d69e2e` — signature color, used sparingly on CTA,
  focus rings, the readiness percent display, decorative diamonds
- Brass-body `--ra-accent-body` — AA-safe variant for body-size use
- Oxblood (`#7a2222` → `#3a0f0f` gradient) — reserved for the deckbox
  SVG and the pitch-red card frame; NOT a UI accent
- Parchment (`#faf3e0`) — shows only on card face illustrations, never
  as a UI surface
- Pitch frames: red (`#c42d2d`), yellow (`#e8c040`), blue (`#2a6db2`) —
  only on `<CardArt>`, never on UI chrome

**Typography** (Google Fonts, preconnect + `display=swap`):
- **Cinzel** (display, 400/700) — headings, nav, labels with eyebrow
  treatment
- **Cinzel Decorative** (900, reserved) — ONE treatment only:
  `.ra-readiness-display` on deck cards + `ReadinessHero`
- **UnifrakturCook** (fraktur, display-only) — ONE usage: the "R"
  emblem on the deckbox logo mark. Never on body copy.
- **IBM Plex Sans** (body, 400/500/600) — all running text, labels
- **JetBrains Mono** (tabular numerals) — reserved for percentages,
  counts, money

**Motion.** Subtle. Shimmer on skeletons, opacity cross-fades on image
load, 180–240ms easing on hover. No bounce, no elastic, no spring
physics in the app shell. The lightbox tilt (perspective + rotateX/Y on
mousemove) is the single "extra" — motion earns its place there.

**References.** Museum exhibit captions. 19th-century card game rule
booklets. Metallic brass embossing on oxblood leather. A hand-set
letterpress invitation.

**Anti-references.**
- Templated admin dashboards (Chakra/MUI defaults, rounded-corner cards
  with icon + heading + body text, identical grid)
- AI-slop aesthetics: cyan-on-dark, purple→blue gradients, neon glows,
  glass morphism as decoration, sparklines-as-ornaments
- SaaS pastel palettes
- Border-left accent stripes on alerts/callouts (banned in impeccable)
- Gradient text (banned)

## Design Principles

1. **Signature treatments stay signature.** `.ra-readiness-display`
   (Cinzel Decorative 900 brass) is used on `effectivePercent` numbers
   only. Home hero's "cards missing" count uses a distinct
   `.ra-hero-primary-stat` class so the signature is not diluted.
2. **Dark contrast is AA.** All fg/bg pairs on dark theme must clear
   WCAG AA (4.5:1 body, 3.0:1 large). See `docs/design/v1/contrast-matrix.md`.
3. **CSS Modules only in new code.** Zero `style={{}}` on new surfaces.
   Legacy inline styles on untouched files are tolerated until Plan C
   lands the lint rule as an error.
4. **Touch target ≥ 44×44** (R52). Enforced on buttons via
   `min-block-size`/`min-inline-size`.
5. **Focus-visible is 2px brass + 2px gap.** Single declaration:
   `outline: 2px solid var(--ra-accent); outline-offset: 2px`. No
   secondary ring, no shadow-based focus.
6. **`prefers-reduced-motion` is a first-class path.** Every animation
   collapses to a static equivalent under this media query.
7. **Design file is binding.** `docs/design/v1/*.jsx` prototypes are
   the visual reference. The Babel-standalone runtime is NOT copied;
   the structural + typographic choices ARE.
